# MediVision AI

A Dockerized chest X-ray diagnostic assistant combining a CNN classifier,
Grad-CAM explainability, and a RAG pipeline over clinical guidelines to
draft doctor-readable reports — built entirely on free, open-source
components (no paid APIs, no API keys required).

**Pipeline:** Upload chest X-ray → ResNet-50 classifies findings → Grad-CAM
highlights the driving region → ChromaDB + sentence-transformers RAG
retrieves relevant clinical guidance → Ollama (Mistral-7B, running locally)
drafts a structured report.

> ⚠️ **This is a portfolio/research project, not a medical device.** All
> outputs require physician review and are not a substitute for clinical
> diagnosis.

## Folder structure

Each language/tool lives in its own top-level folder — nothing is mixed
into the Python tree:

```
MediVision-AI/
├── python/                    # All application code (Python)
│   ├── model/                 # classifier.py, gradcam.py, train.py
│   ├── rag/                   # build_index.py, retriever.py
│   ├── llm/                   # generate_report.py (Ollama integration)
│   ├── utils/                 # config_loader.py, preprocess.py
│   └── app/                   # streamlit_app.py (the UI)
├── shell/                     # Bash scripts
│   ├── setup.sh                setup: venv + deps + pull Ollama model
│   ├── build_index.sh          rebuild the RAG index
│   └── run.sh                  launch the Streamlit app
├── docker/                    # Docker + Compose (own folder, own language)
│   ├── Dockerfile
│   └── docker-compose.yml
├── config/                    # YAML configuration
│   └── config.yaml             every path/model-name/threshold lives here
├── data/
│   ├── clinical_guidelines/    sample .txt reference docs used by RAG
│   ├── sample_images/          put test chest X-rays here
│   ├── chestxray14/             (you create this — see Dataset section)
│   ├── chroma_db/               generated by build_index.sh
│   └── gradcam_outputs/         generated at runtime
├── requirements.txt
├── .gitignore
├── .dockerignore
└── README.md
```

|-------------------|-------------------------------------|------|
| CNN backbone       | torchvision ResNet-50 (ImageNet)   | Free |

## Setup (local machine, not this sandbox)

Requires Python 3.10+, ~8GB RAM free for Mistral-7B, and internet access to
download packages/models the first time.

```bash
git clone <your-repo-url>
cd MediVision-AI
bash shell/setup.sh
```

This creates a virtualenv, installs `requirements.txt`, and pulls the
Ollama Mistral-7B model if Ollama is already installed. If Ollama isn't
installed yet, install it from https://ollama.com/download first, then
re-run `bash shell/setup.sh`.

### Build the RAG index

```bash
bash shell/build_index.sh
```

This embeds the sample files in `data/clinical_guidelines/` into ChromaDB.
Add your own `.txt` reference documents to that folder and re-run this
script any time you want to refresh the index.

### Start Ollama (separate terminal)

```bash
ollama serve
```

### Run the app

```bash
bash shell/run.sh
```

Then open the URL Streamlit prints (typically http://localhost:8501).

## Fine-tuning the classifier on real data (optional but recommended)

Out of the box, the classifier loads an ImageNet-pretrained ResNet-50 with
an **untrained** classification head — the app will run and the pipeline
will work end-to-end, but predictions won't be clinically meaningful until
you fine-tune it. This is intentional: the model weights are too large to
ship in the repo, and downloading the full dataset requires you to do it
directly.

1. Download the NIH ChestX-ray14 dataset (free, public):
   https://nihcc.app.box.com/v/ChestXray-NIHCC
2. Place it at `data/chestxray14/` so you have:
   ```
   data/chestxray14/images/*.png
   data/chestxray14/Data_Entry_2017.csv
   ```
3. Run training:
   ```bash
   python -m python.model.train
   ```
   This saves the best checkpoint to `python/model/weights/resnet50_chestxray.pth`,
   which `classifier.py` automatically picks up on the next run.

Training config (epochs, batch size, learning rate, split ratios) all lives
in `config/config.yaml` under the `training:` key — no code changes needed
to tune it.

## Running with Docker

```bash
cd docker
docker compose up --build
```

This starts two containers: `ollama` (the LLM runtime) and `app` (the
Streamlit UI), networked together. Ollama's model still needs to be pulled
once inside the running container:

```bash
docker exec -it medivision-ollama ollama pull mistral:7b
```

Then open http://localhost:8501.
